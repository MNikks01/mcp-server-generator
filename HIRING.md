# MCP Server Generator — HIRING

This is the wedge — built mostly solo, fast. Minimal hiring; it funds/feeds the next products. Portfolio principles: [../BUSINESS_STRATEGY.md §5](../BUSINESS_STRATEGY.md).

## Stage 0 — Solo founder ($0)
Build the whole MVP solo in ~6 weeks with AI coding agents against this spec (D-011). This product teaches you MCP deeply — that knowledge powers #1 and #2, so the founder *should* build it personally.

## Stage 1 — First engineer (~$0–10K MRR, often shared with #2)
A full-stack engineer (TS) to maintain the generator + hosting while the founder moves to Codebase Intelligence (#2). Likely the same first hire across the wedge (shared monorepo/stack).

## Stage 2 — (usually deferred to the platform)
Because this product's revenue is secondary to its funnel role, dedicated hiring here is minimal. AI-engineer and DevRel hires happen at the portfolio level (driven by #2/#1 traction), not for this tool alone.

## DevRel note
DevRel is high-leverage for THIS product specifically (it's a developer tool riding a hot standard). When DevRel is hired (portfolio Stage 3), the MCP Generator's OSS + tutorials + demos are a primary surface for them.

## Principles
- Don't staff up a wedge — keep it lean, let PLG/OSS run, harvest the funnel.
- The founder personally owns this to learn MCP (the lab's core competency).
- Reuse the shared team/stack across #3/#2/#1.

## What NOT to do
Don't hire sales for a mostly-self-serve free generator. Don't build a big team around the wedge — its job is momentum + audience + skill, then graduate effort to #2/#1.
